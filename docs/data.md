# Data Reference

This document describes the data structures used and generated by the
pipeline, plus the external data sources.

## ЁЯУе Input structure

### FASTA files (`Inputs/`)
The pipeline accepts standard FASTA files placed in the `Inputs/`
directory. Headers should follow a simple pattern for traceability:

```text
>peptide_001
GIGAVLKVLTTGLPALISWIKRKRQQ
>peptide_002
GIGKFLHSAKKFGKAFVGEIMNS
```

- **Length range**: generally between 5 and 100 amino acids (depends
  on the tool).
- **Amino acids**: the 20 standard amino acids are expected. Sequences
  with ambiguous characters (X, B, Z) may be flagged as invalid by some
  parsers.

## ЁЯУд Output structure (`Outputs/`, Phase 1)

The results of each run are stored in the corresponding folder under
`Outputs/`.

### 1. `consolidated.csv` / `consolidated.xlsx`
Main results matrix. One row per peptide. Schema produced by the
orchestrator:

| Column | Description |
|---|---|
| `structural_score` | Integer score based on the activity profile per category (sum of polarities: POS=3, SPLIT=2, NEG=1, NONE=0 for desirable categories). |
| `structural_max` | Maximum possible structural score given the evaluated categories. |
| `holistic_score` | Continuous aggregate score used for quantitative ranking (tiebreaker for the structural score). |
| `n_categories_evaluated` | Number of categories evaluated for the peptide. |
| `apex_potency_tag` | Qualitative potency label (e.g. `POTENTE_AMP`, `MUY_POTENTE_AMP`). |
| `apex_potency_min_mic_uM` | Minimum MIC (in ┬╡M) observed in any strain by APEX. |
| `peptide_id` | Peptide identifier (taken from the FASTA). |
| `sequence` | Amino-acid sequence. |
| `length` | Sequence length. |
| `<tool>__class` | Binary classification (`positive`, `negative`, or null if there was an error or it does not apply). |
| `<tool>__score` | Probability score emitted by the tool (0.0 to 1.0). |
| `<tool>__<metric>__<unit>` | Additional tool-specific metrics (e.g. `apex__mean_mic_pathogen__uM`). |
| `<category>__consensus` | Inter-tool consensus for a category (`POS`, `NEG`, `SPLIT`, `NONE`). |
| `<category>__mean_score` | Mean score of the tools that contributed to that category. |
| `agreement_<category>` | Agreement level for the category (`consensus_positive`, `consensus_negative`, `split`, `single_tool`, `no_call`). |

### 2. `REPORT.md` / `REPORT.html`
Standalone interactive and static report. Includes:
- Hierarchical ranking of therapeutic viability.
- Per-peptide summary and results matrix.
- Dynamic filters by length and scores (HTML).
- APEX metrics broken down per bacterial strain.
- Potency badges (`MUY POTENTE`, `POTENTE`, `PATHOGEN SPECIFIC`).
- Intra-category disagreements and extra continuous metrics.

### 3. `tool_health_report.json`
Status and performance of each tool during the run:
- `status`: `OK` (execution and parsing succeeded) or `PROBLEMATIC`
  (batch failure).
- `runtime_seconds`: tool execution time.
- `n_batches_ok` / `n_batches_failed`: batch-execution detail.
- `diagnosis`: specific-failure diagnosis.

## ЁЯУП Length handling per tool

Each tool has an optimal length range based on its training set. The
orchestrator handles out-of-range sequences according to the tool's
technical behavior:

| Tool | Optimal range | Failure mode | Technical evidence |
|---|---|---|---|
| `acp_dpe` | `[5, 50]` | **`hard_limit`** | `Test.py:152` (fixed embedding) |
| `apex` | `[5, 100]` | **`soft_truncate`** | `predict.py:44` (truncates to 52) |
| `deepb3p` | `[2, 50]` | **`soft_truncate`** | `amino_acid.py:78` (truncates to 50) |
| `toxinpred3`| `[1, 35]` | `soft_reliability` | Based on training set |
| `hemodl` | `[2, 133]` | `soft_reliability` | Based on training set |
| `hemopi2` | `[2, 100]` | `soft_reliability` | Based on training set |

## ЁЯФН Leakage grades (scientific confidence)

> **Phase 2 feature** тАФ applied only in the scientific audit flow
> (`bin/audit_pipeline.sh`), not in the user inference report. See
> `docs/leakage_analysis.md` for the methodology.

To avoid inflated results from sequences already known to the models,
the pipeline classifies peptides into 4 novelty tiers:

- **ЁЯЯв Gold (high novelty)**: sequences that survive CD-HIT-2D at 40%
  identity vs. the training set. Most trustworthy predictions.
- **ЁЯЯб Silver (medium novelty)**: similarity between 40% and 60%.
- **ЁЯЯа Bronze (low novelty)**: similarity between 60% and 80%.
- **ЁЯФ┤ Red (leaked/known)**: similarity >80%. The model probably
  already "knows" this sequence (or a close one), so its score is
  biased.

## ЁЯЧДя╕П Reference databases

The pipeline uses data from the following sources to build evaluation
pools and detect leakage:

| DB | Main content |
|---|---|
| **UniProt** | Sequences annotated with bioactivity and taxonomic origin. |
| **DBAASP** | Antimicrobial peptides with experimental MIC data. |
| **APD3** | Antimicrobial Peptide Database. |
| **ConoServer** | Cone-snail toxins. |
| **ArachnoServer** | Spider toxins. |

Raw data from these sources is processed and stored under
`DATABASES_FASTA/`, then consolidated into
`Dataset_Bioactividad/Category_Pools/`.
